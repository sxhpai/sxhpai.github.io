C语言——复习记录1

1、在计算机内存中，整数一律采用补码形式来存储

2、小数的长度是固定的，float始终占4个字节，double始终占8个字节

3、wchar_t一种“数据类型”

  • 在微软编译器下，它的长度是2个字节，等价于unsinged short
  • 在GCC/LLVM/Clang下，它的长度是4个字节，等价于unsigned int

4、C语言字符编码

  • 对于 char 类型的窄字符，始终使用 ASCII 编码。
  • 对于 wchar_t 类型的宽字符和宽字符串，使用 UTF-16 或者 UTF-32 编码，它们都是基于 Unicode 字符集的。
  • 对于 char 类型的窄字符串，微软编译器使用本地编码，GCC、LLVM/Clang 使用和源文件编码相同的编码。

5、C语言的输入输出与缓冲区

  • 对于输出操作，清空缓冲区会使得缓冲区中的所有数据立即显示到屏幕上；很明显，这些数据没有地方存放了，只能输出了。
  • 对于输入操作，清空缓冲区就是丢弃残留字符，让程序直接等待用户输入，避免引发奇怪的行为。

    清空缓冲区方法：将输入缓冲区中的数据都读取出来。

6、C语言的数组是静态的，大小不可变

7、数组的越界和溢出

  • C语言数组是静态的，不能自动扩容，当下标小于零或大于等于数组长度时，就发生了越界。
  • C语言为了提高效率，保证操作的灵活性，并不会对越界行为进行检查，即使越界了，也能够正常编译，只有在运行期间才可能会发生问题。
  • 由于C语言的”放任“，我们访问数组时必须非常小心，要确保不会发生越界。

8、递归

  • 要想让递归函数逐层进入再逐层退出，需要解决两个方面的问题:
    • 存在限制条件，当符合这个条件时递归便不再继续。对于 factorial()，当形参 n 等于 0 或 1 时，递归就结束了。
    • 每次递归调用之后越来越接近这个限制条件。对于 factorial()，每次递归调用的实参为 n - 1，这会使得形参 n 的值逐渐减小，越来越趋近于 1 或 0。

  • 尾递归，中间递归，多层递归，烧脑程度逐级上升。

  • 递归的时间开销极大，尽量用 迭代 来替换递归函数
